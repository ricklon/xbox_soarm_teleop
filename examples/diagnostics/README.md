# Diagnostics

These scripts are still useful, but they are not part of the primary teleop
entry-point surface.

- Use these for calibration, troubleshooting, controller probing, or low-level
  robot inspection.
- Prefer the top-level scripts in [`examples/`](../README.md) for normal
  teleoperation, simulation, and recording workflows.
- When a packaged CLI exists, prefer it over the wrapper script path:
  `xbox-joint-diagnostic`, `analyze-joint-diag`, `diagnose-robot`, `joint-rom-test`.
